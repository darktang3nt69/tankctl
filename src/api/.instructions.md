# Backend API Layer

You're working in the TankCtl **API layer**. This is where REST endpoints and request/response validation live.

## Use the Specialized Agent

For deep expertise in **FastAPI endpoint design**, **Pydantic validation**, and **request/response schemas**, invoke:

```
/backend-core
```

### When to Use `/backend-core`

- Designing new REST endpoints
- Creating Pydantic request/response schemas
- Implementing validation and error handling
- Coordinating with services and repositories
- Optimizing endpoint performance

### Example Prompts

- `/backend-core Design a new REST endpoint for device firmware updates`
- `/backend-core Create a Pydantic schema for water schedule configuration`
- `/backend-core Add pagination support to the device list endpoint`

## Architecture Reminder

Strict layering in TankCtl:

```
API (this layer)
  ↓ (via dependency injection)
Services (business logic)
  ↓
Repository (data access)
  ↓
Infrastructure (DB, MQTT)
```

**CRITICAL**: Never access the database or MQTT directly from routes. Always use services.
