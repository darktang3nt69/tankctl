# Domain Layer

You're working in the **domain layer**. This is where pure, framework-agnostic business entities and logic live.

## Domain Models Are Pure

Domain models should:
- ✅ Use only Python standard library
- ✅ Contain no framework dependencies (no SQLAlchemy, FastAPI, Pydantic in this layer)
- ✅ Be immutable or use clear state transitions
- ✅ Represent business concepts (Device, Command, Shadow, etc.)

❌ DON'T: Import from SQLAlchemy, FastAPI, or infrastructure

## Example Domain Model

```python
from dataclasses import dataclass
from datetime import datetime
from typing import Optional

@dataclass
class Device:
    """Pure domain model - no imports from framework."""
    device_id: str
    name: str
    location: Optional[str] = None
    created_at: Optional[datetime] = None
    
    def is_online(self, timeout_seconds: int = 300) -> bool:
        """Business logic: determine if device is online."""
        # Pure computation, no side effects
        return True
```

## Separation of Concerns

**Domain Layer:**
```
- Device
- DeviceShadow  
- Command
- Telemetry
- Event
- Alert
```

**Infrastructure Models:**
```
- DeviceModel (SQLAlchemy)
- DeviceSchema (Pydantic)
```

## Conversion Flow

```
Pydantic Schema (input)
    ↓ (validate)
Domain Model (business logic)
    ↓ (persist)
SQLAlchemy Model (database)
```
