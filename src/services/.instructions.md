# Backend Service Layer

You're working in the **service layer**. This is where business logic lives and coordinates different components.

## Use the Specialized Agents

Depending on what you're implementing, choose the right specialist:

### For Core Backend Services
**API design, database queries, repository patterns:**
```
/backend-core
```

When to use:
- Implementing device service methods
- Adding repository access patterns
- Coordinating service orchestration
- Example: `/backend-core Design a new service method for device state management`

### For Device Communication & Commands
**MQTT, device shadow, command execution:**
```
/device-communication
```

When to use:
- Implementing shadow reconciliation
- Managing command execution
- Handling device acknowledgments
- Example: `/device-communication Implement automatic retry logic for failed commands`

### For Notifications & Alerts
**FCM, alert rules, reminders:**
```
/notifications-and-alerts
```

When to use:
- Implementing alert service methods
- Adding reminder scheduling logic
- Managing user notification preferences
- Example: `/notifications-and-alerts Design water-low alert thresholds and FCM delivery`

## Service Architecture

Services orchestrate repositories, domain models, and infrastructure:

```python
class DeviceService:
    def __init__(self, device_repo: DeviceRepository):
        self.repo = device_repo
    
    async def create_device(self, device_id: str, name: str) -> Device:
        # Business logic + validation
        if await self.repo.get_by_id(device_id):
            raise ValueError("Device exists")
        return await self.repo.create(Device(device_id, name))
```
