# Device Communication Infrastructure

You're working in the **MQTT infrastructure layer**. This is where device-backend communication happens.

## Use the Specialized Agent

For deep expertise in **MQTT topics**, **device shadow reconciliation**, and **command protocols**, invoke:

```
/device-communication
```

### When to Use `/device-communication`

- Designing MQTT topic hierarchies
- Implementing pub/sub message handlers
- Creating device shadow reconciliation logic
- Managing command versioning and idempotency
- Handling device acknowledgments and retries
- Writing device firmware protocols

### Example Prompts

- `/device-communication Design the MQTT topic structure for a new sensor type`
- `/device-communication Implement device shadow reconciliation for light scheduling`
- `/device-communication Create an idempotent command handler for pump control`

## MQTT Topic Standard

All topics follow: `tankctl/{device_id}/{channel}`

**Standard Channels:**
```
tankctl/{device_id}/telemetry      → Device → Backend (sensor data)
tankctl/{device_id}/reported       → Device → Backend (state reports)
tankctl/{device_id}/command        → Backend → Device (commands)
tankctl/{device_id}/status         → Device → Backend (status updates)
```
