# Flutter State Management

You're working in the **providers layer**. This is where Riverpod state management and dependency injection are configured.

## Use the Specialized Agent

For deep expertise in **Riverpod providers**, **state management**, and **dependency injection**, invoke:

```
/flutter-foundation
```

### When to Use `/flutter-foundation`

- Creating new Riverpod providers
- Setting up async data providers with caching
- Implementing ChangeNotifier ViewModels
- Configuring dependency injection
- Handling provider invalidation and refreshes
- Manager error states and loading states

### Example Prompts

- `/flutter-foundation Create a Riverpod provider for device telemetry with caching`
- `/flutter-foundation Implement a ViewModel provider for water schedule management`
- `/flutter-foundation Design a dependency injection setup for the device repository`

## Riverpod Provider Patterns

**Service providers (singletons):**
```dart
final deviceServiceProvider = Provider((ref) => DeviceService());
```

**Async data providers:**
```dart
final deviceListProvider = FutureProvider<List<Device>>((ref) async {
  final repo = ref.watch(deviceRepositoryProvider);
  return repo.getAllDevices();
});
```

**ViewModel providers:**
```dart
final deviceViewModelProvider = ChangeNotifierProvider((ref) {
  return DeviceViewModel(ref.watch(deviceRepositoryProvider));
});
```

## Current Providers

- Service providers: API client, authentication
- Repository providers: Device, telemetry, alert data access
- ViewModel providers: Feature-specific state management
- UI state providers: Theme, navigation, app config
