# Flutter Feature Architecture

You're working in the **features layer**. This is where complete features are built using MVVM pattern with clear separation of concerns.

## Use the Specialized Agent

For deep expertise in **MVVM architecture**, **widget testing**, **navigation**, and **component design**, invoke:

```
/flutter-foundation
```

### When to Use `/flutter-foundation`

- Implementing feature screens (Views)
- Creating ViewModel state management
- Writing widget and integration tests
- Designing reusable feature widgets
- Setting up feature navigation
- Managing feature data flow

### Example Prompts

- `/flutter-foundation Create a ViewModel for the device detail screen`
- `/flutter-foundation Write widget tests for the tank card component`
- `/flutter-foundation Design the navigation flow for water scheduling`

## Feature Structure

Each feature follows MVVM:

```
features/device_detail/
├── data/
│   ├── models/              # API models
│   ├── repositories/        # Concrete implementations
│   └── datasources/         # API/local storage
├── domain/
│   ├── entities/            # Domain models (immutable)
│   ├── repositories/        # Abstract interfaces
│   └── usecases/            # Business logic (if needed)
└── presentation/
    ├── viewmodels/          # ChangeNotifiers with state
    ├── widgets/             # Reusable components
    ├── screens/             # Full screens
    └── pages/               # Page-level views
```

## Current Features

- `dashboard/` — Main device list and overview
- `tank_detail/` — Device telemetry and control
- `events/` — Alert and notification history

## Views (Screens) Should Be "Dumb"

Views only contain:
- Layout logic
- Simple conditional rendering (show/hide widgets)
- Animation logic tied to screen size
- Routing logic

**Never** put business logic, API calls, or state management in Views.

## ViewModels Contain Business Logic

```dart
class DeviceDetailViewModel extends ChangeNotifier {
  final DeviceRepository _repo;
  Device? device;
  bool isLoading = false;
  
  Future<void> loadDevice(String deviceId) async {
    isLoading = true;
    notifyListeners();
    try {
      device = await _repo.getDevice(deviceId);
    } finally {
      isLoading = false;
      notifyListeners();
    }
  }
}
```
